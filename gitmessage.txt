# Git Commit Message Template (<type>: <subject>)


# 제목(Subject)은 50자 이내로, 대문자로 시작하며 마침표 없이 작성합니다.
# 한 줄 비우고 본문을 작성할 수 있습니다.
# 본문은 선택사항이며, 변경 이유, 영향, 참고 사항 등을 기록합니다.

# 자주 사용하는 타입(Type)
# feat     : 새로운 기능 추가
# fix      : 버그 수정
# docs     : 문서 수정
# style    : 코드 포맷, 세미콜론 누락 등 (코드 변화 없음)
# refactor : 리팩토링 (기능 변화 없음)
# test     : 테스트 코드 추가/수정
# chore    : 빌드 업무, 패키지 매니저 설정 등
# ci       : CI 관련 설정 변경
# perf     : 성능 개선

# 예시:
# feat: 로그인 API 구현
#
# OAuth2 기반 로그인 처리 로직 추가
# - 사용자 프로필 저장 기능 구현
# - 인증/인가 로직 분리
#
# 관련 이슈: #17
